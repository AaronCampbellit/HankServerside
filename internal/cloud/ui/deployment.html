Hank Remote

Deployment Guide

Run the cloud and agent together on the same machine under one Docker Compose stack, then expose the cloud service externally through Cloudflare Tunnel.

Production Notes

Keep TLS in front of the cloud service, persist PostgreSQL on durable storage, and rotate agent tokens with overlap instead of hard cutovers.

Topology

iPhone App → HTTPS/WebSocket → Hank Remote Cloud → outbound WebSocket → Hank Remote Agent

The cloud is the only public entrypoint. The agent stays inside the home network and connects outbound.

Required Cloud Environment

HANK_REMOTE_CLOUD_ADDR
Bind address for the cloud server.
HANK_REMOTE_CLOUD_DATABASE_URL
PostgreSQL connection URL for the cloud service.
HANK_REMOTE_SESSION_TTL_SECONDS
App-session lifetime.
HANK_REMOTE_REQUEST_TIMEOUT_SECONDS
Relay timeout for app commands.

Required Agent Environment

HANK_REMOTE_AGENT_CLOUD_URL
Public ws:// or wss:// /ws/agent URL.
HANK_REMOTE_AGENT_ID
Stable agent identity.
HANK_REMOTE_AGENT_TOKEN
Issued agent token.
HANK_REMOTE_AGENT_HOME_NAME
Friendly name shown in registration.

Smoke Checklist

1. /healthz returns 200.
2. /readyz reports storage ready.
3. /metrics exposes Prometheus text.
4. One app session authenticates and creates a home.
5. One agent registers and appears online.
6. One system.ping round-trip succeeds.
